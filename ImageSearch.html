Fashion visual search

Why Photo Finder

Finding affordable clothes from images of luxurious clothing found online can require hours of scrolling through different clothing retailers. I built a visual search system that allows users to upload an image of a clothing item and find similar products across multiple retailers. I built a custom web scraper to create a large dataset of fashion product images and metadata, trained a CLIP-based model to create fashion-specific embeddings, indexed the embeddings with FAISS for fast retrieval, and built an iOS workflow that allows users to take pictures of clothing items and find similar products across multiple retailers.

Retail Catalog

Built custom scrapers for Amazon, Zara, ASOS, SHEIN, Macy's, and Temu, collecting product titles, image URLs, prices, metadata, and category information into JSON, CSV, and NDJSON datasets.

Fashion Embeddings

Trained and fine-tuned CLIP-based fashion embeddings with product images and DeepFashion-style clothing data which is a dataset of 80,000 labeled clothes images. I curated an algorithm using contrastive similar/dissimilar image pairs so the model learned fashion-specific similarity.

Fast Retrieval

Converted catalog images into 512-dimensional vector embeddings with the fine-tuned CLIP model and indexed them with FAISS for fast nearest-neighbor search across tens of thousands of products.

Improved Matching

Combined CLIP semantic embeddings with color histograms and shape/HOG features using weighted fusion, coarse retrieval, reranking, category filters, and evaluation stages to improve result quality.

Backend + Mobile Flow

Built Flask APIs, serialized FAISS indexes, product metadata, unified retailer catalogs, and an iOS/mobile workflow that sends a camera image to the backend and returns closest matches with similarity scores.

AWS Deployment

Deployed the Python ML stack as an AWS-backed backend service so the iOS app could keep model inference off-device and meet Apple platform requirements. Provisioned an Ubuntu EC2 t3.large instance, configured the fashion-search API environment, installed Python dependencies, and hosted the FAISS/vector-search service behind a public domain for network image-search requests.

Cloud Architecture

Used S3 buckets to organize product images, vectorized catalog artifacts, model/index files, and deployment resources separately from the application runtime. The REST API loaded serialized FAISS indexes and product metadata, accepted client image uploads, generated embeddings, searched the indexed catalog, and returned the closest fashion products.
